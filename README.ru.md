## Установка

```bash
# Шаг 1: Добавьте маркетплейс
/plugin marketplace add https://github.com/CodeAlive-AI/claude-code-reflection-skills.git

# Шаг 2: Установите плагин
/plugin install claude-code-reflection-skills@claude-code-reflection-skills

# Шаг 3: Перезапустите Claude Code для применения изменений
```

## Что входит в комплект

Плагин **claude-code-reflection-skills** включает 7 навыков:

| Навык | Описание |
|-------|----------|
| **claude-mcp-manager** | Подключение к базам данных, GitHub и API через MCP-серверы |
| **claude-hooks-manager** | Автоформатирование, автотесты, логирование команд |
| **claude-settings-manager** | Настройка прав доступа, песочницы, выбора модели |
| **claude-subagents-manager** | Создание специализированных агентов под конкретные задачи |
| **skills-manager** | Организация и распространение навыков между проектами |
| **claude-plugins-manager** | Упаковка и публикация собственных плагинов |
| **optimizing-claude-code** | Аудит репозиториев и оптимизация CLAUDE.md |

> **Легковесный:** Описания всех 7 навыков занимают менее 500 токенов в контекстном окне.

## Примеры использования

### claude-mcp-manager

> Подключение к базам данных, GitHub и API через MCP-серверы

**Подключение к базе данных**
> *«Подключи Claude к моей PostgreSQL базе»*

Устанавливает [MCP-сервер для баз данных](https://github.com/modelcontextprotocol/servers), настраивает строку подключения. Теперь можно работать с данными в диалоговом режиме.

**Интеграция с GitHub**
> *«Подключи Claude к GitHub, чтобы он мог создавать PR и работать с issues»*

Устанавливает [официальный GitHub MCP-сервер](https://github.com/github/github-mcp-server). Claude сможет создавать ветки, пул-реквесты и работать с задачами напрямую.

**Обновление API-ключей**
> *«Обнови мой API-ключ CodeAlive в конфиге MCP»*

Редактирует `~/.claude.json` или `.mcp.json` для обновления ключей. Идеально для ротации учётных данных без переустановки.

**Массовое обновление конфигурации**
> *«Установи таймаут 120 секунд для всех моих MCP-серверов»*

Массово обновляет конфигурацию MCP-серверов. Можно менять таймауты, переменные окружения и аргументы команд сразу для всех серверов.

### claude-hooks-manager

> Автоформатирование, автотесты, логирование команд

**Автоформатирование кода**
> *«Запускай Prettier для TypeScript-файлов после каждого редактирования»*

Добавляет хук PostToolUse. Каждый файл, который редактирует Claude, автоматически форматируется — никаких разночтений в стиле.

**Автозапуск тестов**
> *«Запускай pytest при редактировании Python-файлов»*

Добавляет хук PostToolUse для `*.py` файлов. Мгновенная обратная связь о том, не сломали ли изменения что-нибудь.

**Логирование команд**
> *«Записывай все команды Claude в файл аудита»*

Добавляет хук PreToolUse, который дописывает команды в `~/.claude/command-log.txt`. Полная история действий.

**Блокировка опасных команд**
> *«Добавь хук, который блокирует глобальные rm -rf команды»*

Добавляет хук PreToolUse, отклоняющий деструктивные команды до их выполнения. Защита от случайной потери данных.

**Запрос подтверждения**
> *«Добавь хук, который спрашивает разрешение при командах с 'db reset'»*

Добавляет хук PreToolUse, который приостанавливает выполнение и запрашивает подтверждение для сброса базы данных или других критичных операций.

---

### claude-settings-manager

> Настройка прав доступа, песочницы, выбора модели

**Включение песочницы**
> *«Включи режим песочницы, чтобы Claude мог работать без запроса разрешения на каждую команду»*

Активирует [нативную песочницу](https://www.anthropic.com/engineering/claude-code-sandboxing) — сокращает запросы разрешений на 84%, сохраняя безопасность системы.

**Блокировка доступа к секретам**
> *«Запрети Claude читать .env файлы и папку /secrets»*

Добавляет запрещающие правила в permissions. Конфиденциальные файлы остаются защищёнными, даже если Claude попытается к ним обратиться.

**Смена модели**
> *«Используй модель Opus для этого проекта»*

Обновляет настройки для использования `claude-opus-4-5` — улучшенное мышление для сложных архитектурных решений.

**Общая конфигурация для команды**
> *«Настрой проектные settings, чтобы у всех была одинаковая конфигурация Claude»*

Создаёт `.claude/settings.json` в области проекта. Один коммит — и вся команда получает идентичные настройки.

### claude-subagents-manager

> Создание специализированных агентов под конкретные задачи

**Создание код-ревьюера**
> *«Создай субагента-ревьюера, который может только читать файлы и использует Opus»*

Создаёт [кастомного субагента](https://code.claude.com/docs/en/sub-agents) с `tools: Read, Grep, Glob` и `model: opus`. Тщательные, глубокие код-ревью.

**Создание раннера тестов**
> *«Создай субагента, который запускает тесты и сообщает об ошибках»*

Создаёт специализированного агента для запуска тестов с ограниченным набором инструментов для безопасности.

---

### skills-manager

> Организация и распространение навыков между проектами

**Список доступных навыков**
> *«Покажи все мои установленные скиллы»*

Выводит скиллы из пользовательской и проектной областей с их триггерами и описаниями.

**Перемещение навыков между скоупами**
> *«Перенеси этот навык в user-scope, чтобы использовать везде»*

Перемещает навыки между проектной и пользовательской областями для более широкой или узкой доступности.

---

### claude-plugins-manager

> Упаковка и публикация собственных плагинов

**Создание плагина**
> *«Создай новый плагин с моими навыками»*

Создаёт структуру нового плагина с манифестом, README и директориями для навыков.

**Публикация на GitHub**
> *«Опубликуй мой плагин на GitHub»*

Упаковывает плагин и создаёт релиз на GitHub для установки другими пользователями.

---

### optimizing-claude-code

> Аудит репозиториев и оптимизация CLAUDE.md

**Аудит готовности проекта**
> *«Проверь, хорошо ли этот репозиторий настроен для Claude Code»*

Запускает комплексный аудит: анализирует CLAUDE.md файлы, настройки, конфигурацию MCP и структуру проекта. Возвращает приоритизированные рекомендации (P0/P1/P2) с конкретными предложениями по улучшению.

**Оптимизация CLAUDE.md**
> *«Проверь мой CLAUDE.md и предложи улучшения»*

Оценивает качество файла памяти: структуру, лаконичность, валидацию @import, наличие обязательных разделов (команды, архитектура, соглашения). Даёт конкретные правки для повышения дружелюбности проекта к агенту.
